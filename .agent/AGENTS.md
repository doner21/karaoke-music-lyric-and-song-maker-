# ⛔ GRAPHIFY GATE — READ THIS BEFORE ANY OTHER FILE

This project has a knowledge graph at `graphify-out/` generated by Graphify.

## MANDATORY STARTUP SEQUENCE (no exceptions)

Before reading **any** source file from `src/`, `server/`, or `electron/`, and before
running **any** exploratory bash command (`find`, `grep`, `ls`, `wc`, `head`), you MUST:

1. `read("graphify-out/wiki/_INDEX.md")` — if it exists
2. Read the relevant community page(s) in `graphify-out/wiki/02_TOP_COMMUNITIES/`
   for whatever subsystem you're investigating
3. Optionally cross-reference `graphify-out/GRAPH_REPORT.md` for suggested
   questions and god nodes

Only after completing steps 1-2 may you read raw source files.

## WHY THIS EXISTS

A Pi extension (`~/.pi/agent/extensions/graphify.ts`) enforces this at the tool
level — it will **block** reads of `src/`, `server/`, and `electron/` files, and
block exploratory bash commands, until you read at least one file from
`graphify-out/`.

Cooperate with the gate: read the wiki first. It has architecture diagrams,
community breakdowns, and narrative explanations that will make your subsequent
source reading faster and more accurate.

## OTHER RULES

- Do NOT run `graphify update .` automatically. The human triggers `/graphify`
  when the codebase has changed significantly.
